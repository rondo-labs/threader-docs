# 本地数据和云端协作

Threader 有两个数据世界：本地项目数据和云端协作数据。

这条边界需要保持清晰。

## 本地项目数据

本地项目数据包括：

- videos
- events
- clips
- drawings
- playlists
- tactical annotations
- 导入的比赛数据

这些数据属于桌面项目工作流。

## 云端协作数据

云端协作数据包括：

- auth
- workspaces
- members 和 invitations
- feed items
- feed comments
- read state

这些数据属于分享和团队讨论层。

## 桥接

发布到 feed 是从本地分析到云端协作的明确桥接动作。

Threader 不应该无意中模糊这两个世界。
