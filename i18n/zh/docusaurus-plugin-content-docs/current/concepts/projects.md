# 项目

项目是 Threader 的本地分析容器。

一场比赛、一节训练课，或一个聚焦的复盘主题，通常对应一个项目。项目会把足球分析对象连接起来，让视频、事件、clips、annotations、drawings 和 playlists 都指向同一组比赛时刻。

## 项目可以包含

- 比赛视频引用
- 导入或手动创建的事件
- clips
- drawings
- tactical annotations
- playlists 和复盘结构
- 本地项目状态

## 项目数据是本地数据

项目数据是 Threader 桌面工作流的基础。

Workspace 成员和 feed 评论等云端协作数据是另一层，只有在你明确分享或发布分析材料时才会连接起来。

## 实用规则

先从项目和视频开始。随着分析变得更结构化，再添加事件、clips、annotations 和 playlists。
