# 缺失媒体

当项目可以打开，但视频不可用时，查看这一页。

项目可能引用本地媒体文件。如果文件被移动、重命名，或位于当前不可用的硬盘上，Threader 可能需要重新连接媒体。

## 先尝试

1. 确认视频文件仍然存在。
2. 如果文件在外接硬盘上，连接该硬盘。
3. 如果可能，恢复原始文件路径。
4. 在 Threader 提示时重新连接媒体。

## 什么仍然安全

媒体缺失不代表项目数据丢失。事件、clips、annotations 和 playlists 仍然可以存在，但视频播放需要媒体文件。
