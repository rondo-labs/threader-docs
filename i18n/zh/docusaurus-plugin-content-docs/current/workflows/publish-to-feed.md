# 发布到 feed

当分析材料已经准备好进入团队讨论时，发布到 feed。

Feed 是 Threader 的协作层。Clips、playlists、notes 和复盘材料会在这里变成共享讨论。

## 什么时候发布

在你有具体内容之后再发布：

- 需要讨论的 clip
- 用于复盘的 playlist
- 需要教练组反馈的战术 note
- 未来可用的 dashboard 或 report item

不要把 feed 当成本地分析起点。先从项目、视频和 clips 开始。

## Workspace 上下文

Feed 属于 workspace。Workspace 成员、邀请、评论和已读状态都是云端协作数据。

你的本地项目数据会保持独立，直到你明确发布或分享某个内容。

## 发布之后

使用 feed 来：

- 让讨论保持连接到分析材料
- 收集团队评论
- 从讨论回到相关视频时刻
- 在 workspace 中跟踪复盘上下文
