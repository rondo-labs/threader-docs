# 快速开始

当你想从一个空项目开始，做出第一个可以复盘的 clip 时，请从这里开始。

Threader 最核心的工作流是：

```text
Project -> Video -> Event / Time Range -> Clip -> Tactical Annotation -> Playlist / Review -> Feed
```

## 1. 创建或打开项目

为这场比赛创建一个新项目，或者从最近项目中打开已有项目。

项目是本地分析容器。它保存比赛视频、事件、clips、drawings、tactical annotations、playlists 和复盘状态。

## 2. 导入比赛视频

在深入分析前，先导入主要比赛视频。视频可用后，你就可以通过播放器和 timeline 在比赛中定位。

如果 Threader 无法稳定读取视频文件，建议转码成标准 H.264 MP4 后再导入。

## 3. 导入或创建事件

如果你已经有来自 provider 或其他流程的事件数据，可以把它导入项目。

如果暂时没有事件数据，也可以直接从视频开始：选择时间范围，然后手动创建 clips。

## 4. 创建第一个 clip

在视频或 timeline 中找到关键时刻，然后通过下面任一方式创建 clip：

- 从选中的时间范围创建
- 从选中的事件创建

Clip 是第一个稳定的分析对象。它可以被调整、标注、放进 playlist，并在之后分享。

## 5. 添加战术意义

当 clip 需要的不只是标题，而是足球语义时，添加 tactical annotation。

先从最小有用结构开始：

- phase
- concept
- outcome
- 必要时加 notes

## 6. 构建复盘 playlist

当多个 clips 指向同一个复盘主题时，把它们组织成 playlist。

标题应该能脱离软件也被理解，例如 `High press recoveries` 或 `Player 8 receiving under pressure`。

## 7. 准备好后再发布

当你已经有值得讨论的 clip、playlist 或复盘内容时，再使用 workspace 和 feed。

Feed 是协作层，不是本地分析的起点。
