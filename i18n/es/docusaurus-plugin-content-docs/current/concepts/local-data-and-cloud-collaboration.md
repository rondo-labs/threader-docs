# Datos locales y colaboración en la nube

Threader tiene dos mundos de datos: datos locales de proyecto y datos de colaboración en la nube.

Mantén clara esta frontera.

## Datos locales de proyecto

Los datos locales de proyecto incluyen:

- videos
- eventos
- clips
- drawings
- playlists
- tactical annotations
- datos de partido importados

Estos datos pertenecen al flujo de proyecto en escritorio.

## Datos de colaboración en la nube

Los datos de colaboración en la nube incluyen:

- auth
- workspaces
- members e invitations
- feed items
- feed comments
- read state

Estos datos pertenecen a la capa de compartir y discusión de equipo.

## El puente

Publicar en feed es el puente explícito desde análisis local hacia colaboración en la nube.

Threader no debe mezclar estos mundos accidentalmente.
