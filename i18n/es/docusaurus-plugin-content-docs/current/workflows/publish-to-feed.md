# Publica en feed

Usa esto cuando el material de análisis esté listo para discusión del equipo.

Feed es la capa de colaboración de Threader. Es donde clips, playlists, notas y material de revisión se convierten en conversación compartida.

## Cuándo publicar

Publica después de tener algo concreto:

- un clip que necesita discusión
- una playlist para revisión
- una nota táctica que necesita feedback del staff
- un dashboard o report item cuando esté disponible

No trates feed como el punto de inicio del análisis local. Empieza con el proyecto, el video y los clips.

## Contexto de workspace

Feed pertenece a un workspace. La membresía, invitaciones, comentarios y estado de lectura son datos de colaboración en la nube.

Tus datos locales de proyecto permanecen separados hasta que publicas o compartes algo explícitamente.

## Después de publicar

Usa feed para:

- mantener la discusión conectada al material de análisis
- recoger comentarios del staff
- volver desde una discusión al momento de video relacionado
- seguir el contexto de revisión dentro de un workspace
