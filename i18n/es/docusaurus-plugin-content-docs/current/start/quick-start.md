# Inicio rápido

Usa esta guía cuando quieras pasar de un proyecto vacío a tu primer clip listo para revisar.

Threader funciona mejor si piensas en este flujo:

```text
Project -> Video -> Event / Time Range -> Clip -> Tactical Annotation -> Playlist / Review -> Feed
```

## 1. Crea o abre un proyecto

Crea un proyecto nuevo para el partido o abre uno existente desde la lista de recientes.

Un proyecto es el contenedor local de análisis. Guarda el video del partido, eventos, clips, drawings, tactical annotations, playlists y el estado de revisión.

## 2. Importa el video del partido

Antes de hacer análisis detallado, importa el video principal del partido. Cuando el video esté disponible, usa el reproductor y el timeline para moverte por el partido.

Si Threader no puede leer el archivo de forma estable, transcodifícalo a un MP4 H.264 estándar e importa esa versión.

## 3. Importa o crea eventos

Si ya tienes datos de eventos de un provider u otro flujo, impórtalos en el proyecto.

Si todavía no tienes eventos, puedes trabajar directamente desde el video seleccionando rangos de tiempo y creando clips manualmente.

## 4. Crea tu primer clip

Encuentra un momento en el video o en el timeline y crea un clip desde:

- un rango de tiempo seleccionado
- un evento seleccionado

Los clips son el primer objeto de análisis duradero. Se pueden ajustar, anotar, agrupar en playlists y compartir más adelante.

## 5. Añade significado táctico

Añade una tactical annotation cuando el clip necesite significado futbolístico más allá de un título.

Empieza con la estructura mínima útil:

- phase
- concept
- outcome
- notes si hace falta

## 6. Construye una playlist de revisión

Agrupa clips relacionados en una playlist cuando quieras preparar material para revisión técnica.

Usa nombres que funcionen fuera de la app, como `High press recoveries` o `Player 8 receiving under pressure`.

## 7. Publica cuando esté listo

Usa workspace y feed después de tener algo que valga la pena compartir, como un clip, una playlist o una nota de revisión.

Feed es la capa de colaboración. No es obligatorio para empezar el análisis local.
