# Contributing

Thanks for being part of the Threader community!

## Reporting Issues

If you run into any problems while using Threader, or have ideas for improvement, please [open an issue](https://github.com/rondo-labs/threader-docs/issues/new).

It helps to include:

- **What you were trying to do** — the match, workflow, or analysis task
- **What happened** — unexpected behavior, confusing UI, missing functionality, etc.
- **Steps to reproduce** — so we can investigate and fix it
- **Screenshots or screen recordings** — if applicable

### Issue Labels

| Label | Use for |
|-------|---------|
| `bug` | Something isn't working as expected |
| `feature-request` | Ideas for new features or improvements |
| `question` | General questions about using Threader |
| `ux` | Feedback on the user interface or workflow |

## Writing a Blog Post

We'd love to hear about your experience using Threader! If you'd like to share a workflow, case study, or tips from your analysis work, you can submit a blog post via pull request.

### How to submit

1. Fork this repository
2. Create a new file in the `blog/` folder, named like `2026-04-01-your-title.md`
3. Use this template:

```markdown
---
slug: your-title
title: Your Blog Post Title
authors:
  - name: Your Name
    url: https://your-link (optional)
tags: [workflow, match-analysis, tips, etc.]
---

Your content here.

<!-- truncate -->

The rest of your post...
```

4. Submit a pull request — we'll review and publish it

### What to write about

- How you use Threader in your daily workflow
- Match analysis case studies
- Tips and tricks you've discovered
- How Threader fits into your data pipeline

## Code of Conduct

Be respectful and constructive. We're all here to make football analysis better.
