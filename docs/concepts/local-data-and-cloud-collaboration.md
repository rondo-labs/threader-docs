# Local Data And Cloud Collaboration

Threader has two data worlds: local project data and cloud collaboration data.

Keep this boundary clear.

## Local Project Data

Local project data includes:

- videos
- events
- clips
- drawings
- playlists
- tactical annotations
- imported match data

This data belongs to the desktop project workflow.

## Cloud Collaboration Data

Cloud collaboration data includes:

- auth
- workspaces
- members and invitations
- feed items
- feed comments
- read state

This data belongs to the sharing and team discussion layer.

## The Bridge

Publishing to feed is an explicit bridge from local analysis to cloud collaboration.

Threader should not blur these worlds accidentally.
