# Rosters and Players

Rosters connect events and clips to the people involved in the match. Threader can store teams, players, lineups, and reusable library records so you do not need to rebuild squads from scratch for every project.

## Where Rosters Are Used

Use player and team records when you:

- Tag an event with the player who performed the action
- Link clips to one or more players
- Filter the timeline by player
- Import match data from a provider file
- Export CDF data with complete metadata

## Player Tags

Player tags can be attached to events and clips. This keeps the timeline and clip library useful for review sessions where one moment may involve several players.

## Library Records

Threader includes roster library flows so teams and players can be reused across projects. This is useful for clubs and analysts who work with the same squad repeatedly.
