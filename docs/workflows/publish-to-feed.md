# Publish To Feed

Use this when analysis material is ready for team discussion.

Feed is Threader's collaboration layer. It is where clips, playlists, notes, and review material become shared conversation.

## When To Publish

Publish after you have something concrete:

- a clip that needs discussion
- a playlist for review
- a tactical note that needs staff feedback
- a dashboard or report item when available

Do not treat feed as the starting point for local analysis. Start with the project, video, and clips first.

## Workspace Context

Feed belongs to a workspace. Workspace membership, invitations, comments, and read state are cloud collaboration data.

Your local project data remains separate until you explicitly publish or share something.

## After Publishing

Use feed to:

- keep discussion attached to analysis material
- collect staff comments
- return from a discussion to the related video moment
- track review context across a workspace
