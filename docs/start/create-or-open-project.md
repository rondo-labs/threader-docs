# Create Or Open A Project

Use this when you are starting a match analysis session.

A Threader project is the local workspace for one analysis context. It keeps video references, imported or manual events, clips, drawings, tactical annotations, playlists, and project state together.

## Create A New Project

1. Start Threader.
2. Choose the new project action.
3. Name the project after the match, training session, or review topic.
4. Save it somewhere your analysis files are allowed to live.

Use one project per match when the analysis depends on match video, match time, or event data.

## Open An Existing Project

Use recent projects when returning to active analysis work.

If a project opens without its media, the project file is still intact; Threader may need you to reconnect missing video files.

## What Belongs In A Project

- match video references
- event data
- clips
- drawings
- tactical annotations
- playlists and review structure
- local analysis state

## What Does Not Belong Only To A Project

Workspace membership, feed comments, invitations, and account profile information belong to the collaboration layer.

Projects are local analysis containers. Workspaces and feed are explicit sharing surfaces.
